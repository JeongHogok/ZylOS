# Zyl OS Emulator

웹 브라우저에서 Zyl OS를 테스트할 수 있는 디바이스 에뮬레이터입니다.
하드웨어 없이 앱 개발과 OS 기능을 체험할 수 있습니다.

## 빠른 시작

```bash
# 프로젝트 루트에서 실행
python3 -m http.server 9000

# 브라우저에서 접속
# http://localhost:9000/emulator/index.html
```

## 디바이스 프로필

| 프로필 | 네비게이션 | RAM | 화면 |
|--------|----------|-----|------|
| Zyl Phone F3 | 제스처 바 (iOS 스타일) | 16GB | 1080×2400 |
| Zyl Phone F3 Lite | 소프트키 3버튼 | 8GB | 1080×2340 |
| Zyl Phone F3 Classic | 물리 하드웨어 버튼 | 4GB | 720×1280 |

## 조작법

### 제스처 바 모드
- 바를 위로 끝까지 드래그 → 홈
- 바를 위로 중간까지 → 최근 앱
- 바를 좌/우로 → 앱 전환

### 상태바
- 상태바 클릭 → 퀵설정 패널 열기/닫기

### 키보드 단축키
- `H` — 홈
- `Esc` — 뒤로가기
- `R` — 최근 앱
- `L` — 잠금
- `P` — 전원

### 잠금화면
- 위로 드래그 → PIN 입력 (기본 PIN: 0000)

## 시스템 서비스

에뮬레이터는 실제 OS의 D-Bus 서비스를 postMessage IPC로 시뮬레이션합니다:
- 파일 시스템, WiFi, Bluetooth, 기기 정보, 저장공간
- 배터리: 호스트 PC의 실제 배터리 API 사용
- 스토리지: 호스트 브라우저의 실제 스토리지 쿼터 사용

## 앱 테스트

에뮬레이터에 포함된 앱: 홈, 잠금화면, 설정, 브라우저, 파일, 터미널, 카메라

커스텀 앱 테스트: [앱 개발 가이드](../docs/APP_DEVELOPMENT_GUIDE.md) 참고

## 구조

```
index.html       — 디바이스 프레임 + 제어 패널
js/emulator.js   — 앱 라우팅, 네비게이션, 알림
js/services.js   — 시스템 서비스 시뮬레이션
js/hal-browser.js — 호스트 하드웨어 API 연동
css/emulator.css — 디바이스 프레임 스타일
```

## 제한사항

- 카메라: 시뮬레이션 (실제 카메라 미사용)
- 전화/SMS: 시뮬레이션 (VoIP 미지원)
- 브라우저: 에뮬레이터 내 실제 URL 로딩 불가 (디바이스에서만 가능)
